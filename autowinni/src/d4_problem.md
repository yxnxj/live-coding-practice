# D4 Java — 작업 명령 처리

- 권장 제한시간: 30분
- 풀이 파일: `D4Solution.java`

## 메서드 형태

```java
public static List<String> processCommands(List<String> commands) {
    // 구현
}
```

## 문제

작업을 추가하거나 실행하는 명령이 입력 순서대로 주어진다. 각 명령을 순서대로 처리하고, 실제로 실행된 작업 ID를 실행 순서대로 반환하라.

## 명령 형식

각 명령은 다음 두 형식 중 하나다.

```text
ADD 작업ID
RUN
```

정상 입력의 명령은 다음 조건을 만족한다.

1. `ADD`와 `RUN`은 항상 대문자로 주어진다.
2. `ADD` 뒤에는 공백 한 칸과 작업 ID가 온다.
3. 작업 ID는 공백이 없는 비어 있지 않은 문자열이다.
4. 같은 작업 ID가 여러 번 추가될 수 있으며, 각각 별개의 작업으로 취급한다.

개별 명령의 형식이 올바른지는 검사하지 않아도 된다.

## 처리 규칙

1. `ADD 작업ID`를 만나면 해당 작업을 아직 실행되지 않은 작업들의 마지막에 추가한다.
2. `RUN`을 만나면 아직 실행되지 않은 작업 중 가장 먼저 추가된 작업 하나를 실행한다.
3. 작업을 실행하면 그 작업 ID를 결과 목록의 마지막에 추가한다.
4. 실행할 작업이 없는 상태에서 `RUN`을 만나면 아무 일도 하지 않는다.
5. 모든 명령을 확인한 뒤 실행되지 않은 작업은 결과에 포함하지 않는다.

## 반환 규칙

1. 실제로 실행된 작업 ID를 실행 순서대로 담은 새 `List`를 반환한다.
2. `commands`가 `null`이거나 비어 있으면 빈 `List`를 반환한다.
3. 원본 `commands`를 변경하지 않는다.

## 입력 크기

`commands`의 길이는 최대 `200,000`이다.

## 예제

```java
List<String> commands = List.of(
    "ADD JOB-100",
    "ADD JOB-200",
    "RUN",
    "ADD JOB-300",
    "RUN",
    "RUN",
    "RUN"
);
```

처리 과정:

```text
ADD JOB-100  → JOB-100 추가
ADD JOB-200  → JOB-200 추가
RUN          → JOB-100 실행
ADD JOB-300  → JOB-300 추가
RUN          → JOB-200 실행
RUN          → JOB-300 실행
RUN          → 실행할 작업이 없으므로 무시
```

기대 결과:

```java
List.of("JOB-100", "JOB-200", "JOB-300")
```

## 추가로 확인할 조건

- 첫 명령이 `RUN`인 경우
- `ADD`만 있고 `RUN`이 없는 경우
- 여러 작업을 추가한 뒤 연속으로 실행하는 경우
- 작업 추가와 실행이 번갈아 등장하는 경우
- 같은 작업 ID가 여러 번 추가되는 경우
- `commands`가 `null`이거나 빈 목록인 경우
