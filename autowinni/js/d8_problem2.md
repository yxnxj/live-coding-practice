# D8 JavaScript 2 — 다음으로 더 높은 가격

- 제한시간: 20분
- 풀이 파일: `d8_problem2.js`

## 함수 형태

```javascript
function distancesToHigherPrice(prices) {
  // 구현
}
```

## 문제

날짜 순서대로 기록된 차량 가격 배열 `prices`가 주어진다.

각 날짜마다 그 날짜보다 가격이 **엄격하게 높은** 첫 번째 미래 날짜를 찾고, 며칠 뒤인지 반환하라. 이후에 더 높은 가격이 없다면 `0`을 사용한다.

배열의 인덱스를 날짜로 보고 두 인덱스의 차이를 날짜 차이로 계산한다.

## 선택 규칙

1. 현재 인덱스보다 뒤에 있는 가격만 확인한다.
2. 현재 가격보다 엄격하게 큰 가격을 찾는다.
3. 조건을 만족하는 미래 가격이 여러 개면 가장 가까운 인덱스를 선택한다.
4. 같은 가격은 더 높은 가격으로 보지 않는다.
5. 조건을 만족하는 가격이 없으면 `0`을 반환한다.

## 입력 조건

1. `prices`는 정수 배열이며 비어 있을 수 있다.
2. 각 가격은 `1` 이상 `1,000,000` 이하이다.
3. 배열의 길이는 최대 `200,000`이다.
4. 잘못된 입력 형식은 고려하지 않아도 된다.

## 반환 규칙

1. 각 가격에 대한 날짜 차이를 `prices`와 같은 순서로 담은 새 배열을 반환한다.
2. 마지막 원소의 결과는 항상 `0`이다.
3. 빈 배열이 주어지면 빈 배열 `[]`을 반환한다.
4. 원본 `prices`를 변경하지 않는다.

## 예제

```javascript
distancesToHigherPrice([4, 3, 5, 5, 2, 6]);
```

각 인덱스의 결과:

```text
0번 가격 4 → 2번 가격 5: 2
1번 가격 3 → 2번 가격 5: 1
2번 가격 5 → 5번 가격 6: 3
3번 가격 5 → 5번 가격 6: 2
4번 가격 2 → 5번 가격 6: 1
5번 가격 6 → 이후에 더 높은 가격 없음: 0
```

기대 결과:

```javascript
[2, 1, 3, 2, 1, 0]
```

## 추가로 확인할 조건

- 가격이 계속 오르는 경우
- 가격이 계속 내리는 경우
- 모든 가격이 같은 경우
- 같은 가격 뒤에 더 높은 가격이 등장하는 경우
- 더 높은 가격이 멀리 떨어져 있는 경우
- 원소가 하나뿐이거나 없는 경우
- 원본 배열 보존
