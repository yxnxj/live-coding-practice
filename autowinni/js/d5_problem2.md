# D5 JavaScript 2 — 정렬된 배열에 들어갈 위치들

- 권장 제한시간: 25분
- 풀이 파일: `d5_problem2.js`

## 함수 형태

```javascript
function findInsertIndices(numbers, targets) {
  // 구현
}
```

## 문제

오름차순으로 정렬된 정수 배열 `numbers`와 여러 목표값을 담은 배열 `targets`가 주어진다. 각 목표값을 `numbers`에 추가한 뒤에도 오름차순이 유지되도록 삽입할 수 있는 가장 작은 인덱스를 `targets`의 순서대로 반환하라.

실제로 `numbers`에 값을 추가하지는 않으며, 각 목표값은 모두 원래의 `numbers`를 기준으로 판단한다.

## 위치 결정 규칙

각 `target`의 위치는 다음 규칙으로 결정한다.

1. `target`보다 작은 값들은 삽입 위치보다 앞에 있어야 한다.
2. `target`보다 크거나 같은 값들은 삽입 위치부터 뒤에 있어야 한다.
3. `target`과 같은 값이 이미 여러 개 있다면 그중 첫 번째 값의 인덱스를 사용한다.
4. 모든 값이 `target`보다 작으면 `numbers.length`를 사용한다.
5. 모든 값이 `target`보다 크거나 같으면 `0`을 사용한다.

## 입력 조건

정상 입력은 다음 조건을 만족한다.

1. `numbers`는 오름차순으로 정렬된 정수 배열이다.
2. `numbers`에는 중복값과 음수가 포함될 수 있다.
3. `numbers`의 길이는 최대 `1,000,000`이다.
4. `targets`는 정수 배열이며 길이는 최대 `100,000`이다.

정상 입력에서 개별 원소가 정수인지와 `numbers`가 정렬되어 있는지는 검사하지 않아도 된다.

## 반환 규칙

1. 각 목표값의 삽입 인덱스를 `targets`와 같은 순서로 담은 새 배열을 반환한다.
2. `numbers`가 빈 배열이면 각 목표값의 결과는 모두 `0`이다.
3. `targets`가 빈 배열이면 빈 배열 `[]`을 반환한다.
4. 원본 `numbers`와 `targets`를 변경하지 않는다.

## 예제

```javascript
const numbers = [1, 3, 3, 5, 8];
const targets = [3, 4, 0, 10];
```

목표값별 결과:

```text
3  → 1
4  → 3
0  → 0
10 → 5
```

기대 결과:

```javascript
[1, 3, 0, 5]
```

## 추가로 확인할 조건

- 목표값이 이미 존재하는 경우
- 같은 값이 여러 번 존재하는 경우
- 배열의 처음, 중간, 끝에 들어가는 경우
- `numbers`가 비어 있는 경우
- `targets`가 비어 있는 경우
- 같은 목표값이 여러 번 주어지는 경우
- 원본 배열 보존
