# D5 JavaScript 1 — 여러 구간의 합계 조회

- 권장 제한시간: 25분
- 풀이 파일: `d5_problem1.js`

## 함수 형태

```javascript
function rangeTotals(values, queries) {
  // 구현
}
```

## 문제

정수 배열 `values`와 여러 개의 조회 구간을 담은 배열 `queries`가 주어진다. 각 조회 구간에 포함되는 값의 합을 조회 순서대로 담은 새 배열을 반환하라.

## 조회 구간 형식

각 조회는 다음 형태다.

```javascript
[start, end]
```

- `start`와 `end`는 `values`의 0부터 시작하는 인덱스다.
- `start`와 `end` 위치의 값도 합계에 포함한다.
- 정상 입력에서는 항상 `0 <= start <= end < values.length`를 만족한다.

## 입력 조건

정상 입력은 다음 조건을 만족한다.

1. `values`는 정수 배열이며 비어 있을 수 있다.
2. 각 값은 `-1,000,000` 이상 `1,000,000` 이하이다.
3. `values`의 길이는 최대 `200,000`이다.
4. `queries`는 올바른 조회 구간을 담은 배열이다.
5. `queries`의 길이는 최대 `200,000`이다.
6. 모든 구간 합은 JavaScript의 안전한 정수 범위 안에 있다.

정상 입력에서 개별 값과 개별 조회 형식은 검사하지 않아도 된다.

## 반환 규칙

1. 각 조회 구간의 합을 `queries`와 같은 순서로 담은 새 배열을 반환한다.
2. `values` 또는 `queries`가 빈 배열이면 빈 배열 `[]`을 반환한다.
3. 원본 `values`와 `queries`를 변경하지 않는다.

## 예제

```javascript
const values = [5, -2, 3, 7, 1];
const queries = [
  [0, 2],
  [1, 3],
  [3, 3],
  [0, 4]
];
```

각 조회 결과:

```text
[0, 2] → 5 + (-2) + 3 = 6
[1, 3] → -2 + 3 + 7 = 8
[3, 3] → 7
[0, 4] → 5 + (-2) + 3 + 7 + 1 = 14
```

기대 결과:

```javascript
[6, 8, 7, 14]
```

## 추가로 확인할 조건

- 원소 하나만 조회하는 경우
- 배열 전체를 조회하는 경우
- 음수가 포함된 구간
- 같은 구간이 여러 번 등장하는 경우
- 조회가 하나뿐인 경우
- 빈 배열
- 원본 배열 보존
