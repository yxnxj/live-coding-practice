# 03 Tennis

## Type

Branch cleanup / Domain rules

## Practice Goal

- 복잡한 조건문을 읽기 좋은 도메인 규칙으로 바꾼다.
- 점수 상태를 명확한 이름으로 표현한다.
- 작은 문제에서도 과한 설계를 피한다.

## Source

- https://codingdojo.org/kata/Tennis/

## 원본 문제 요약

단순화된 테니스 한 게임의 점수를 구현한다.

- 각 플레이어의 기본 점수는 `love`, `15`, `30`, `40` 중 하나다.
- `40`인 플레이어가 포인트를 따면 보통 게임에서 승리한다.
- 양쪽이 `40`이면 `deuce`다.
- `deuce`에서 한 플레이어가 포인트를 따면 그 플레이어가 `advantage`를 가진다.
- advantage를 가진 플레이어가 다시 포인트를 따면 게임에서 승리한다.
- advantage가 없는 플레이어가 포인트를 따면 다시 `deuce`가 된다.
- 다른 표현으로는 최소 4포인트 이상, 상대보다 2포인트 이상 앞서면 게임에서 승리한다.

## Done Criteria

- 듀스, 어드밴티지, 승리 조건 테스트가 있다.
- 조건문이 의도를 드러내는 이름으로 정리되어 있다.
